traces

supercharged ee tools

ai for the eda. source parts in plain english, catch schematic bugs erc can't, and pull datasheets — inside kicad and your terminal. open source, runs on your machine.

three tools

  • mcp — describe a part (“100nF 0402 X7R”, “low-Rds N-MOSFET SOT-23”, or an mpn) and get ranked, in-stock candidates across jlcpcb, digi-key, and mouser. install it into claude code, codex, or any mcp agent.
  • kicad extension — a kicad 10 plugin that sources parts into your schematic, reconciles metadata, pulls symbols/footprints from lcsc, and fetches datasheets.
  • semantic rule check (src) — erc checks your wiring. src checks the meaning: net-label typos, inconsistent buses, orphan nets, near-duplicate names. run /src from claude code or codex.

local-first

no accounts, no billing, no telemetry. point it at openrouter, or run fully offline with a local model via ollama. live stock and pricing from the suppliers — the model selects, it never invents. mit licensed.

quick start

# clone and configure
git clone https://github.com/usetraces/traces.git
cd traces && cp .env.example .env   # add a key, or use ollama

# run the local sourcing server
cd mcp && ./install.sh && uv run traces-serve

# install the kicad plugin
cd ../kicad-extension && ./install.sh

full setup, the /src skill, and the local-model guide are in the readme.
